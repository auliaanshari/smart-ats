| Kriteria                    | Deskripsi                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| :-------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Dataset**                 | Dataset Resume/CV yang memuat data teks deskripsi profil kandidat dan dilabeli ke dalam 24 kategori profesi yang berbeda (seperti HR, Designer, Information-Technology, dll).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Masalah**                 | Departemen _Human Resources_ (HR) seringkali kewalahan harus menyortir, membaca, dan mengkategorikan ratusan hingga ribuan CV pelamar secara manual. Proses ini sangat memakan waktu, tidak efisien, dan rentan terhadap bias subjektif atau _human error_.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Solusi machine learning** | Membangun sebuah sistem _Smart Applicant Tracking System_ (ATS) berbasis _Natural Language Processing_ (NLP) menggunakan _pipeline_ TensorFlow Extended (TFX). Sistem ini secara otomatis akan membaca teks mentah dari CV pelamar dan memprediksi ke mana kategori profesi (dari 24 kelas) kandidat tersebut paling cocok ditempatkan.                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Metode pengolahan**       | Data diproses secara _end-to-end_ menggunakan TFX. Diawali dengan penyerapan data menggunakan `CsvExampleGen`. Data kemudian divalidasi dan dianalisis menggunakan `StatisticsGen`, `SchemaGen`, dan `ExampleValidator`. Transformasi teks (seperti _lowercasing_, penghapusan karakter khusus/tanda baca, dan _text vectorization_) dilakukan di dalam graf TensorFlow menggunakan komponen `Transform` agar pra-pemrosesan tidak terputus saat tahap _deployment_ (menghindari _training-serving skew_).                                                                                                                                                                                                                                                                                      |
| **Arsitektur model**        | Menggunakan arsitektur _Deep Learning_ khusus NLP yang tangguh. Melalui proses _Hyperparameter Tuning_ menggunakan metode **Bayesian Optimization** (10 trials), ditemukan arsitektur terbaik: _Text Vectorization_ diikuti _Embedding layer_ (dimensi 128), kemudian diekstraksi polanya menggunakan `Conv1D` (128 filters) untuk menangkap frasa (_n-grams_), dan diteruskan ke `Bidirectional LSTM` (32 units) untuk memahami konteks kalimat panjang. Untuk mencegah _overfitting_ yang ekstrem pada data teks yang kecil, diterapkan _L2 Regularization_ (1e-05) dan _SpatialDropout1D_. Lapisan _output_ menggunakan _Dense layer_ dengan aktivasi _Softmax_ untuk klasifikasi 24 kelas. _Loss function_ yang digunakan adalah `sparse_categorical_crossentropy` dengan _optimizer_ Adam. |
| **Metrik evaluasi**         | Metrik utama yang digunakan selama pelatihan adalah `Sparse Categorical Accuracy`. Pada tahapan produksi, evaluasi mendalam (_slicing_) dilakukan oleh komponen `Evaluator` (TFMA) dengan melihat `SparseCategoricalAccuracy` dan memastikan model baru (_candidate_) melampaui performa model sebelumnya (_baseline_).                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Performa model**          | Model berhasil mencapai batas performa optimal data (_data ceiling_) dengan akurasi _Training_ sebesar **~87.9%** dan akurasi _Validation_ sebesar **~66.5%**. Mengingat sistem membedakan 24 kelas profesi dari _dataset_ yang relatif kecil, akurasi >66% menunjukkan model sukses menangkap pola bahasa yang sangat spesifik. Model berstatus _BLESSED_ oleh Evaluator dan sukses di-_deploy_ menggunakan **TensorFlow Serving (Docker)**. Pengujian _request_ klien (`auliaanshari-testing.ipynb`) untuk kategori spesifik seperti _Information-Technology_ mendapatkan tingkat keyakinan (_confidence_) hingga **94.5%**.                                                                                                                                                                  |

## ✨ Fitur Ekstra

Proyek ini mengimplementasikan 3 kriteria tambahan untuk penilaian maksimal:

1. **Hyperparameter Tuning:** Menggunakan komponen `Tuner` (KerasTuner) dengan metode _Bayesian Optimization_ untuk mencari kombinasi _Embedding_, _Conv1D_, dan _LSTM_ secara otomatis.
2. **Model Deployment:** Model yang lulus evaluasi diekspor dan dilayani menggunakan _container_ **TensorFlow Serving** via REST API (Port 8501).
3. **Prediction Script:** Terdapat skrip `auliaanshari-testing.ipynb` (dan _testing cell_ di Notebook) yang membungkus teks CV ke format `tf.train.Example`, melakukan _Base64 encoding_, dan mengirimkan POST _request_ ke server Docker secara otomatis.
